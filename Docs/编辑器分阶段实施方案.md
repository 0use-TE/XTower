# XTower 编辑器 — 需求整理与分阶段实施方案

> 版本：v0.1  
> 定位：类似保卫萝卜的塔防 **内容编辑器**（Avalonia）+ **游戏运行时**（Phaser3 / Web）  
> 协作模型：工作区 = Git 仓库根目录，Content 与配置全部入库  

---

## 一、总体架构

```
┌─────────────────────────────────────────────────────────────┐
│  XTower Editor (Avalonia / C#)                               │
│  · 工作区管理 · 地图 · 怪物/炮塔/音乐 · 精灵动画 · 预览      │
└──────────────────────────┬──────────────────────────────────┘
                           │ 读写同一目录
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  {{Workspace}}/Content/          ← Git 跟踪                 │
│    schemas/  monsters/  turrets/  levels/  music/  assets/  │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP / 文件路径
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  game/ (Phaser3 + TypeScript)   ← 纯逻辑，无 GUI             │
│  · 读 Content · vibe coding 游戏规则                        │
└─────────────────────────────────────────────────────────────┘
```

**原则**

| 原则 | 说明 |
|------|------|
| 数据单一来源 | 编辑器只写 `Content/`，游戏只读，不维护第二份配置 |
| 作者格式中立 | PNG + JSON，非 Phaser 专有二进制 |
| Ava 重 UI，Web 重逻辑 | 编辑器不做游戏规则；游戏不做表单界面 |
| 工作区即仓库 | 打开 Git 根目录即可协作；`.xtower/` 存本地/个人偏好 |

## 二、工作区（Workspace）

### 2.1 启动流程

1. **首次启动** / 菜单「文件 → 打开工作区」→ 选择文件夹（通常为 Git 仓库根）
2. 校验：是否存在 `Content/` 或允许初始化
3. **初始化向导**（目录不存在或 `Content/.xtower-manifest.json` 缺失时）：
   - 创建标准目录树（见 2.3）
   - 写入默认 `schemas/`
   - 写入示例怪物/炮塔/关卡（可选）
4. 记录最近工作区列表 → `%LocalAppData%/XTower/recent-workspaces.jsonl`
5. 进入主界面；标题栏显示 `XTower — {仓库名}`

### 2.2 工作区配置文件

| 文件 | 位置 | 入库 | 用途 |
|------|------|------|------|
| `Content/.xtower-manifest.json` | 工作区 | ✅ | 版本、Content 结构版本、默认关卡 |
| `.xtower/settings.jsonl` | 仓库根 | ❌ gitignore | 个人编辑器偏好（可选覆盖布局） |
| `Content/project.json` | Content | ✅ | 项目元数据：名称、设计分辨率、网格默认值 |

**`Content/.xtower-manifest.json` 示例**

```json
{
  "manifestVersion": 1,
  "contentVersion": 1,
  "defaultLevel": "levels/level01.json",
  "designResolution": { "width": 1920, "height": 1080 },
  "grid": { "cellSize": 32, "columns": 32, "rows": 18 }
}
```

> **网格 vs 视口**：逻辑格固定 **32×32**（炮塔/障碍物/图块一格格对齐）。默认 **32 列 × 18 行** → 世界尺寸 **1024×576**（16:9）。`designResolution` 是运行时 **Camera 缩放目标**，Phaser 将世界矩形拉伸/适配到 1920×1080 视口；背景图也可独立拉伸，不必与格数整除。

### 2.3 初始化目录树

```
{Workspace}/
  Content/
    .xtower-manifest.json
    project.json
    schemas/
      monster.schema.json
      turret.schema.json
      level.schema.json
      music.schema.json
      sprite-atlas.schema.json
    monsters/
      _index.json              # id 列表，编辑器快速加载
    turrets/
      _index.json
    levels/
      _index.json
    music/
      _index.json
    obstacles/                 # 阶段后期
      _index.json
    assets/
      monsters/                # 按实体 id 分子目录
        {monsterId}/
          sheet.png            # AI 生成的精灵条/图集
          atlas.json             # 帧矩形（可编辑器生成）
          preview/               # 可选：AI 输出的单帧序列
            walk_00.png ...
      turrets/{turretId}/
      tiles/                   # 地图图块
      ui/
      audio/
        bgm/
        sfx/
  game/                        # Phaser 工程（可同仓库）
    public/content -> ../Content  或 build 时 copy
  .gitignore                   # 忽略 .xtower/、bin/、node_modules/
  Docs/
```

### 2.4 工作区服务（代码层）

- `IWorkspaceService`：当前路径、是否已初始化、Watch 文件变化
- `IContentRepository`：按类型 CRUD JSON、维护 `_index.json`
- 所有面板 ViewModel 依赖 `IWorkspaceService`；**未打开工作区时**主界面显示「打开/创建工作区」占位页

## 三、AI 序列帧 — 资产格式规范

> 目标：AI 直接产出文件，编辑器 **导入 → 自动/半自动切帧 → 绑定 walk/death** → Phaser 可读。

### 3.1 推荐 AI 输出方式（按优先级）

#### 方案 A：横向精灵条（**最推荐，AI 最好出**）

```
Content/assets/monsters/slime/source/walk_strip.png
Content/assets/monsters/slime/source/death_strip.png
```

| 约束 | 值 |
|------|-----|
| 格式 | **PNG**，RGBA，sRGB |
| 排列 | 单行动画，从左到右等宽帧 |
| 每帧尺寸 | 相同 `frameWidth × frameHeight`（如 128×128） |
| 帧数 | walk 4~8 帧，death 4~6 帧 |
| 背景 | 透明或纯品红 `#FF00FF`（编辑器可抠） |
| 命名 | `{anim}_{index}.png` 或 `{anim}_strip.png` |

编辑器导入时：输入帧宽/高 → **自动生成 `atlas.json`**，无需 AI 输出 JSON。

#### 方案 B：已切好单帧序列

```
Content/assets/monsters/slime/source/walk_00.png … walk_07.png
Content/assets/monsters/slime/source/death_00.png … death_05.png
```

编辑器：批量导入 → 合并为 `sheet.png` + `atlas.json`（打包一步，可缓存）。

#### 方案 C：单张网格图集（AI 较少用）

```
sheet.png  # 2 行 × 4 列，每格同尺寸
```

编辑器：指定行列数 → 切 atlas。

**不推荐 AI 直接出**：TexturePacker 专有格式、Spine 二进制、Phaser Pack File。

### 3.2 编辑器生成的标准产物（每个实体）

```
Content/assets/monsters/{id}/
  sheet.png          # 合并后的图集（walk+death 可各一张或合并）
  atlas.json         # 帧名 → 矩形，见 3.3
  anim.json          # walk/death 帧序列 + 帧率
```

**`atlas.json`（与 Phaser `load.atlas` 兼容的 JSON Hash 格式）**

```json
{
  "frames": {
    "walk_0": { "frame": { "x": 0, "y": 0, "w": 128, "h": 128 } },
    "walk_1": { "frame": { "x": 128, "y": 0, "w": 128, "h": 128 } }
  },
  "meta": {
    "image": "sheet.png",
    "size": { "w": 1024, "h": 128 },
    "scale": "1"
  }
}
```

**`anim.json`**

```json
{
  "walk":  { "frames": ["walk_0","walk_1","walk_2","walk_3"], "frameRate": 8,  "repeat": -1 },
  "death": { "frames": ["death_0","death_1","death_2"],       "frameRate": 10, "repeat": 0 }
}
```

### 3.3 实体配置引用（`monsters/slime.json`）

```json
{
  "id": "slime",
  "displayName": "史莱姆",
  "sprite": {
    "atlas": "assets/monsters/slime/atlas.json",
    "animations": "assets/monsters/slime/anim.json",
    "defaultAnimation": "walk"
  },
  "stats": { "hp": 100, "speed": 80, "armor": 0, "goldReward": 10 },
  "behaviorId": "slime_basic"
}
```

炮塔同理：`turrets/cannon.json` + `assets/turrets/cannon/`。

### 3.4 音乐资产

| 类型 | 格式 | 路径 |
|------|------|------|
| BGM | OGG 优先，MP3 备选 | `assets/audio/bgm/{id}.ogg` |
| SFX | OGG / WAV | `assets/audio/sfx/{id}.ogg` |

`music/forest.json`：

```json
{
  "id": "forest_bgm",
  "type": "bgm",
  "path": "assets/audio/bgm/forest.ogg",
  "loop": true,
  "volume": 0.8
}
```

AI 生成音乐：导出 **WAV/MP3** → 编辑器可选转 OGG（阶段后期）；起步直接支持 MP3。

### 3.5 AI 生成清单（给美术/AI 的 Prompt 模板）

```
为塔防游戏生成 {entityName} 的 {walk|death} 序列帧：
- 输出：横向 PNG 精灵条
- 每帧：128×128，共 {N} 帧，透明背景
- 风格：{style ref}
- 方向：面向右（或等距 3/4）
- 死亡动画：最后一帧可停在倒地状态
```

## 五、数据 Schema 概要

所有 `Content/schemas/*.schema.json` 用于编辑器校验；实体 JSON 人类可读、Git diff 友好。

| Schema | 文件位置 |
|--------|----------|
| monster | `Content/monsters/*.json` |
| turret | `Content/turrets/*.json` |
| level | `Content/levels/*.json` |
| music | `Content/music/*.json` |
| sprite-atlas | `assets/**/atlas.json` + `anim.json` |

**level.json 核心字段（自定义，不强制 Tiled）**

```json
{
  "id": "level01",
  "name": "第一关",
  "grid": { "columns": 32, "rows": 18, "cellSize": 32 },
  "paths": [
    { "id": "path-1", "name": "主路", "color": "#F44336", "waypoints": [{ "col": 0, "row": 8 }, { "col": 31, "row": 8 }] }
  ],
  "obstacles": [{ "x": 5, "y": 5, "tileId": "rock_01" }],
  "initialTowers": [{ "turretId": "cannon", "x": 3, "y": 7 }],
  "waves": [],
  "meta": { "startingGold": 200, "startingLives": 20, "bgmId": "forest_bgm" }
}
```

## 六、分阶段实施方案

### Phase 0：基础壳（当前 → 可打开工作区）

**目标**：能选文件夹、初始化 Content、空面板有占位提示。

| 任务 | 说明 |
|------|------|
| P0-1 | `IWorkspaceService` / `WorkspaceService` |
| P0-2 | 启动对话框：打开最近 / 浏览目录 |
| P0-3 | `InitializeWorkspace()` 创建 2.3 目录树 + manifest |
| P0-4 | 未打开工作区时 MainView 显示 Welcome 页 |
| P0-5 | 标题栏、状态栏显示工作区路径 |
| P0-6 | `.gitignore` 模板写入建议（`.xtower/`） |
| P0-7 | `IContentRepository` 读 `_index.json` |

**交付**：打开 Git 仓库 → 自动/init Content → 项目树可见。

---

### Phase 1：Content CRUD + 项目树

**目标**：怪物/炮塔/音乐/关卡的 JSON 列表与新建删除。

| 任务 | 说明 |
|------|------|
| P1-1 | `schemas/monster.schema.json` 等首版 |
| P1-2 | ProjectView：TreeView 绑定 Content 真实目录 |
| P1-3 | 新建向导：输入 id → 生成空 JSON + 更新 index |
| P1-4 | 删除 / 重命名（防呆：引用检查占位） |
| P1-5 | Monster/Turret/Music View：仅列表 + 空表单占位 |
| P1-6 | `DataPersistence` 与工作区路径统一（布局仍 LocalAppData，Content 在工作区） |

**交付**：在编辑器里新建 `slime` 怪物条目，磁盘出现 JSON。

---

### Phase 2：精灵导入与 walk/death 动画（AI 序列帧）
